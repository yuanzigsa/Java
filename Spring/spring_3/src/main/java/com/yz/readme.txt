用户登录
    整体思路：
        1.接受参数（用户名、密码）
        2.判断参数（参数的非空判断）
        3.通过用户名查询用户记录，返回用户对象
        4.判断用户对象是否为空
        5.如果用户对象不为空，判断密码是否正确
        6.如果密码正确，则登录成功

        Ccontroller层：（控制层：接收请求、响应结果）
            1.接受参数
            2.调用业务逻辑层对应的方法，得到登录结果（ResultInfo对象）
            3.响应结果
        Serveice层：（业务逻辑层：判断数据、数据处理）
            1.参数判断（判断参数是否为空）
                如果参数为空，设置ResultInfo对象，并返回
            2.调用Dao层的查询方法，通过用户名查询用户记录，返回用户对象
            3，判断用户对象是否为空
                如果用户对象为空，设置ResultInfo对对象，并返回
            4.判断密码是否正确（判断传递的密码与数据中查询的密码是否相等）
                如果密码不正确，设置ResultInfo对象，并返回
            5.登录成功，设置ResultInfo对象，并返回
        Dao层：（数据访问层：数据库的增删改查）
            通过用户名查询用户记录，返回用户对象
